# Submission Checklist

Please ensure your submission includes all of the following:

## Required
- Source code
- `README.md`
- `AI_USAGE.md`
- evaluation note
- output or results for the seed questions
- at least 5 additional self-designed test questions
- short walkthrough

## README Must Cover
- how to install and run the project
- assumptions made
- architecture summary
- retrieval, tool-use, or routing approach
- trade-offs you made intentionally
- limitations and known failure modes
- what you would improve next

## Evaluation Note Must Cover
- how you measured quality
- what metrics or criteria you used
- examples of successful behavior
- examples of failure or uncertainty
- how you would regression-test future changes

## AI Usage Disclosure Must Cover
- what tools you used
- where they helped
- where you overrode or corrected them
- what decisions were ultimately yours

## Submission Format
A Git repository or compressed folder is acceptable.

## Optional But Helpful
- tests
- sample commands or screenshots
- brief architecture diagram
- logs or traces showing how your system made a decision
